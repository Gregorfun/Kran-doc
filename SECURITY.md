# Security Policy

## Supported Versions

Aktuell wird die `main`-Version aktiv gepflegt.

## Vulnerability Reporting

Bitte keine Sicherheitslücken öffentlich posten.

Stattdessen:

- **kein öffentliches Issue erstellen**
- direkt melden per Mail (oder private Nachricht)

## Sicherheitshinweise

- Keine echten Kundendaten committen
- `.env` Dateien niemals ins Repo hochladen
- PDF-Dateien vor Verarbeitung prüfen
- OCR-Quellen validieren

## Geplante Verbesserungen

- Input-Validierung für Uploads
- Rate-Limiting
- Authentifizierungssystem
- Logging & Monitoring
